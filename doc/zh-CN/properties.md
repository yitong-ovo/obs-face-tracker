# 参数说明

## 识别与跟踪的区别

人脸/人体检测大约每两秒运行一次。两次检测之间由 dlib correlation tracker
逐帧跟踪目标，所以画面中的检测框和亮绿色跟踪框更新频率不同。

## 识别方式

### 经典人脸检测（dlib HOG）

适合近景、正脸和 CPU 资源有限的场景。调试框为红色虚线。

### 深度人脸检测（dlib CNN）

旧版深度检测器，通常最慢。只有它确实比其他模式准确时才建议使用。调试框
为橙色点线。

### 广角增强检测（YuNet + NanoDet）

推荐用于广角、全身和小脸场景：

- YuNet 直接识别人脸，黄色实线。
- NanoDet 识别人体，蓝色虚线。
- YuNet 没有提供对应人脸时，根据人体顶部估算脸框，绿色点线。
- 真实 YuNet 人脸优先于估算脸。
- 蓝色人体框仅用于调试，不会直接作为人脸 tracker 的初始化框。

## 模型路径

| 设置 | 文件 |
|------|------|
| 经典人脸模型 | `frontal_face_detector.dat` |
| 深度人脸模型 | `mmod_human_face_detector.dat` |
| YuNet 小脸模型 | `face_detection_yunet_2023mar.onnx` |
| NanoDet 人体模型 | `nanodet-plus-m_416.onnx` |

Release 包一般会自动设置。只有路径为空、使用自定义模型或本地编译时才需要
手工选择。

## 检测下采样倍率（Scale image）

这是下采样倍率，不是放大倍率：

| 数值 | 效果 |
|------|------|
| `1` | 原始分辨率，最容易识别小脸，CPU 最高 |
| `2` | 宽高各缩小一半，推荐默认值 |
| 大于 `2` | CPU 更低，但小脸更容易漏检 |

## 目标框扩展

**目标框向左/右/上/下扩展**决定识别人脸周围包含多少额外区域，会影响最终
构图和 tracker 初始化。

## 检测区域裁剪

**检测时忽略左侧/右侧/顶部/底部像素**只限制周期性检测区域。已经开始的
相关跟踪可以暂时移动到检测区之外。

## 目标位置和画面大小

- **目标大小（Zoom）**：控制人物在输出画面中的大小。
- **目标水平位置（X）**：负数偏左，正数偏右。
- **目标垂直位置（Y）**：负数偏上，正数偏下。
- **最大放大倍数（Scale max）**：限制最大裁切放大。
- **输出画面比例（Aspect）**：输出宽高比。

## 跟随稳定性

- **跟随速度（Kp）**：越大响应越快，也更容易抖动。
- **持续偏差修正（Ki）**：修正长期无法居中的偏差。
- **移动预测（Td）**：预测目标移动趋势。
- **移动/缩放平滑时间（LPF）**：过滤快速变化。
- **防抖范围（Dead band）**：范围内的小误差不触发移动。
- **柔和跟随范围（Nonlinear band）**：从静止到跟随平滑过渡。

建议先保持默认值，只在确认检测框稳定但输出画面仍过快、过慢或抖动时调整。

## 调试设置

- **在预览中显示识别框**：显示检测与 tracker 框。
- **只显示识别结果，不移动画面**：用于检查检测区域和构图空间。
- **始终显示识别框**：节目输出中也显示，主要用于演示，不建议直播时误开。
